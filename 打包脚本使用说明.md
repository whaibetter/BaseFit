# BaseFit 自动化打包脚本使用说明

## 📦 脚本功能

`build-release.ps1` 是一个功能强大的PowerShell自动化打包脚本，用于Android APK的版本管理和发布构建。

### 主要特性

- ✅ **交互式命令行界面** - 友好的用户交互
- ✅ **版本号自动管理** - 支持语义化版本规范
- ✅ **自动版本递增** - 默认自动递增修订号
- ✅ **灵活的输出路径** - 支持绝对路径和相对路径
- ✅ **自动构建流程** - 一键执行完整的Gradle构建
- ✅ **版本信息生成** - 自动创建version.txt文件
- ✅ **完善的错误处理** - 验证输入、检查路径、捕获错误
- ✅ **参数模式支持** - 支持命令行参数直接指定

---

## 🚀 快速开始

### 方式一：交互模式（推荐新手使用）

直接运行脚本，按提示输入即可：

```powershell
.\build-release.ps1
```

### 方式二：参数模式（适合自动化）

通过命令行参数直接指定版本号和输出路径：

```powershell
# 指定版本号
.\build-release.ps1 -Version 2.1.0

# 指定版本号和输出路径
.\build-release.ps1 -Version 2.1.0 -Output .\release

# 使用绝对路径
.\build-release.ps1 -Version 1.5.0 -Output C:\Releases\BaseFit
```

## 📋 版本号规范

### 语义化版本控制（Semantic Versioning）

版本号格式：`MAJOR.MINOR.PATCH`

- **MAJOR（主版本号）**: 不兼容的API修改
- **MINOR（次版本号）**: 向下兼容的功能性新增
- **PATCH（修订号）**: 向下兼容的问题修正

### 有效版本号示例

| 版本号 | 说明 |
|--------|------|
| `1.0.0` | 初始版本 |
| `1.0.1` | Bug修复 |
| `1.1.0` | 新功能添加 |
| `2.0.0` | 重大更新 |

### 无效版本号示例

- ❌ `1.0` - 缺少修订号
- ❌ `v1.0.0` - 包含前缀v
- ❌ `1,0,0` - 使用逗号分隔
- ❌ `1.x.0` - 包含字母

## 📁 输出路径

### 默认输出路径

如果不指定输出路径，脚本会自动使用：

```
项目根目录/dist/版本号/
```

例如：
```
c:\Users\whaif\Downloads\android\baseFit\dist\1.0.1\
```

### 自定义输出路径

#### 相对路径

```powershell
.\build-release.ps1 -Output .\releases
```
输出到：`项目根目录\releases\`

#### 绝对路径

```powershell
.\build-release.ps1 -Output C:\MyApps\BaseFit
```
输出到：`C:\MyApps\BaseFit\`

### 输出目录内容

打包完成后，输出目录会包含：

```
dist/1.0.1/
├── BaseFit-v1.0.1.apk    # 签名后的APK文件
└── version.txt             # 版本信息文件
```

## 🔧 工作流程详解

### 完整打包流程

1. **版本号处理**
   - 读取当前版本号（从app/build.gradle.kts）
   - 验证版本号格式
   - 自动递增修订号（如果未指定）

2. **更新配置文件**
   - 更新app/build.gradle.kts中的versionName
   - 递增versionCode

3. **执行Gradle构建**
   - 运行 `gradlew.bat clean`
   - 运行 `gradlew.bat assembleRelease`
   - 等待构建完成

4. **复制构建产物**
   - 查找app/build/outputs/apk/release/app-release.apk
   - 重命名为BaseFit-v{版本号}.apk
   - 复制到输出目录

5. **生成版本信息**
   - 创建version.txt
   - 记录版本号、构建时间、Git commit等

6. **完成提示**
   - 显示最终版本号
   - 显示输出路径
   - 显示耗时统计

## ⚠️ 错误处理

### 常见错误及解决方案

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| 版本号格式无效 | 版本号不符合x.y.z格式 | 检查版本号，使用正确格式 |
| 找不到build.gradle.kts | 脚本不在项目根目录运行 | 确保在项目根目录执行脚本 |
| Gradle构建失败 | 代码编译错误 | 检查代码，修复编译错误 |
| 找不到APK文件 | 构建失败或路径错误 | 检查构建是否成功 |
| 输出路径不可写 | 权限不足或路径无效 | 检查路径权限或更换路径 |

### 退出码

| 退出码 | 含义 |
|--------|------|
| 0 | 成功 |
| 1 | 失败 |

## 📝 最佳实践

### 1. 版本管理建议

- **初始版本**: 从 `1.0.0` 开始
- **Bug修复**: 只递增修订号（如1.0.0 → 1.0.1）
- **新功能**: 递增次版本号（如1.0.1 → 1.1.0）
- **重大更新**: 递增主版本号（如1.1.0 → 2.0.0）

### 2. 发布流程建议

1. **测试版本**
   ```powershell
   .\build-release.ps1 -Version 1.0.0-beta.1 -Output .\beta-test
   ```

2. **正式发布**
   ```powershell
   .\build-release.ps1 -Version 1.0.0
   ```

3. **Git标签**
   ```powershell
   git tag -a v1.0.0 -m "Release version 1.0.0"
   git push origin v1.0.0
   ```

### 3. CI/CD集成

可以在GitHub Actions、Jenkins等CI/CD工具中使用：

```yaml
# GitHub Actions 示例
- name: Build Release APK
  shell: powershell
  run: |
    .\build-release.ps1 -Version ${{ github.ref_name }} -Output ./artifacts
    
- name: Upload APK
  uses: actions/upload-artifact@v3
  with:
    name: basefit-apk
    path: ./artifacts/
```
