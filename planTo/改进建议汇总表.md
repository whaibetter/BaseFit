## 功能增强类改进建议

### F1: 打卡编辑/删除功能

| 属性 | 内容 |
|------|------|
| **问题现状** | 用户打卡后无法修改记录，也无法删除错误记录 |
| **改进建议** | 在打卡记录详情页增加编辑和删除按钮 |
| **预期效果** | 数据管理完整性提升，用户误操作可修正 |
| **实施难度** | 低 |
| **涉及文件** | RecordScreen.kt, CheckInScreen.kt, Repository.kt |
| **预计工时** | 4小时 |

### F2: 动作搜索与筛选

| 属性 | 内容 |
|------|------|
| **问题现状** | 动作管理页面无法按名称搜索，无法按分类筛选 |
| **改进建议** | 增加搜索框和分类FilterChip组件 |
| **预期效果** | 查找效率提升约40% |
| **实施难度** | 低 |
| **涉及文件** | ExerciseManagementScreen.kt |
| **预计工时** | 4小时 |

### F3: 计划模板功能

| 属性 | 内容 |
|------|------|
| **问题现状** | 用户每次设置计划都需要手动添加动作 |
| **改进建议** | 创建常用计划模板库，支持模板保存和调用 |
| **预期效果** | 计划设置时间减少60% |
| **实施难度** | 中 |
| **涉及文件** | PlanScreen.kt, Repository.kt, Entities.kt |
| **预计工时** | 8小时 |

### F4: 批量打卡支持

| 属性 | 内容 |
|------|------|
| **问题现状** | 只能逐个动作进行打卡 |
| **改进建议** | 支持多选动作后一键批量打卡 |
| **预期效果** | 操作效率提升50% |
| **实施难度** | 中 |
| **涉及文件** | HomeScreen.kt, ViewModels.kt |

## 技术优化类改进建议

### T1: 数据库索引优化

| 属性 | 内容 |
|------|------|
| **问题现状** | exercises、week_plans等表无索引，查询效率低 |
| **改进建议** | 为高频查询字段添加索引（exerciseId, dayOfWeek, date） |
| **预期效果** | 查询性能提升30%以上 |
| **实施难度** | 低 |
| **涉及文件** | Entities.kt, AppDatabase.kt |
| **预计工时** | 2小时 |

### T2: N+1查询优化

| 属性 | 内容 |
|------|------|
| **问题现状** | getTodayPlans()方法在循环中调用getById，导致N+1问题 |
| **改进建议** | 使用Room的JOIN查询或@Relation注解 |
| **预期效果** | 列表加载速度提升，数据库访问减少90% |
| **实施难度** | 中 |
| **涉及文件** | Repository.kt, Daos.kt |
| **预计工时** | 6小时 |

### T3: 依赖版本升级

| 属性 | 内容 |
|------|------|
| **问题现状** | Compose BOM 2023.10.01、Room 2.6.1等版本较旧 |
| **改进建议** | 升级到最新稳定版本（Compose BOM 2024.02.00+） |
| **预期效果** | 性能优化、新特性、bug修复 |
| **实施难度** | 中 |
| **涉及文件** | build.gradle.kts |
| **预计工时** | 8小时 |

### T4: 引入依赖注入

| 属性 | 内容 |
|------|------|
| **问题现状** | ViewModel和Repository手动创建实例，耦合度高 |
| **改进建议** | 集成Hilt依赖注入框架 |
| **预期效果** | 代码耦合度降低，可测试性提升 |
| **实施难度** | 中 |
| **涉及文件** | ViewModels.kt, Repository.kt, build.gradle.kts |
| **预计工时** | 12小时 |

### T5: 数据加密实现

| 属性 | 内容 |
|------|------|
| **问题现状** | 数据库和文件存储均为明文 |
| **改进建议** | 使用SQLCipher对数据库加密 |
| **预期效果** | 数据安全性大幅提升 |
| **实施难度** | 中 |
| **涉及文件** | AppDatabase.kt, build.gradle.kts |
| **预计工时** | 8小时 |

### T6: 全局异常处理

| 属性 | 内容 |
|------|------|
| **问题现状** | 异常处理分散在各ViewModel中 |
| **改进建议** | 实现统一的全局异常处理器 |
| **预期效果** | 崩溃率降低，错误追踪更便捷 |
| **实施难度** | 中 |
| **涉及文件** | BaseFitApp.kt, ViewModels.kt |
| **预计工时** | 4小时 |
